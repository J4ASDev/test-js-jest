Qué son las pruebas unitarias

La prueba unitarias son un proceso de desarrollo de software en donde se toman todos los bloques de codigo de una aplicación y se examinan individual e independientemente para un funcionamiento adecuado. Así, todo lo que vamos pasando sabemos que esta funcionando correctamente y que no hay ningún inconveniente o bug.

Las características de las pruebas unitarias son:

  • Automatizable: Deben correr sin ningún proceso manual.
  • Total Cobertura: Debemos de pasar por cada bloque escrito.
  • Reutilizables: Podemos usarlas para probar otros bloques.
  • Independientes: No pueden depender de otra prueba para funcionar.
  • Rápidas de crear: Tienen que ser algo conciso que prueben algo muy particular.

Ventajas de las pruebas unitarias:

  • Proporciona un trabajo ágil.
  • Calidad del código.
  • Detectar errores rapido.
  • Facilita los cambios y favorece la integración.
  • Proporciona información.
  • Reduce el coste.

Que es Jest

Jest es una herramienta creada y mantenida por la comunidad open source con apoyo de Facebook. Jest surgió con el objetivo de agregar pruebas unitarias a proyectos con react pero ha sido tan importante que cada vez se emplea en mas proyectos con otros frameworks.

Como estructurar una prueba con Jest

Antes de pasar a la segunda etapa, tenemos que tener en claro la base de Jest (Pruebas unitarias en general), como se estructura una prueba y como funciona un poco el api que nos ofrece Jest.

Veamos 3 palabras que seran vitales en nuestro camino de las pruebas unitarias:

  • describe(): Crea un bloque que agrupa varias pruebas relacionadas. Recibe 2 parametros:
    1. Una descipcion/string: 'Bloque de pruebas'
    2. Una funcion anonima: () => {}

    Resultado: describe('Nuevo bloque de prueba', () => {} )

  • test(): El Método sirve para ejecutar una prueba. Recibe 2 parametros:
    1. Una descipcion/string: 'Bloque de pruebas'
    2. Una funcion anonima: () => {}

    Resultado: test('Primera prueba!', () => {} )

  • expect(): Se utiliza cada vez que desea testear un valor. Recibe 1 parametro:
    1. Un valor con el cual interactuar

Una vez entendido que hacen estas tres palabras podremos crear nuestra primera prueba;

    
      describe('Bloque de pruebas', () => {
          test('Primera prueba', () => {
              expect().toBe();
          });
          test('Segunda prueba', () => {
              expect().toBe();
          });
      });

Comparadores Comunes

Los comparadores comunes nos permiten probar los valores de diferentes maneras.

    Los comparadores comunes pueden ser:
  • toBe():
  • not.toBe()

Ejemplo: 

  
    test('two plus two is four', () => {
      expect(2 + 2).toBe(4);
    });

Numeros

La mayoría de las formas de comparar números tienen equivalentes de matcher.

    Los Métodos para numeros que nos provee jest son:
  • toBeGreaterThan: El valor debe ser mayor a la comparacion.
  • toBeGreaterThanOrEqual: El valor debe ser mayor o igual a la comparacion.
  • toBeLessThan: El valor debe ser menor a la comparacion.
  • toBeLessThanOrEqual: El valor debe ser menor o igual a la comparacion
  
    test('two plus two', () => {
      const value = 2 + 2;
      expect(value).toBeGreaterThan(3);
      expect(value).toBeGreaterThanOrEqual(3.5);
      expect(value).toBeLessThan(5);
      expect(value).toBeLessThanOrEqual(4.5);
    });

Verdadero

En las pruebas, a veces es necesario distinguir entre undefined, null, true o false.

    Los métodos que nos provee Jest para hacer estas validaciones son:
  • .toBeNull(): El valor debe ser nulo/null
  • .toBeTruthy(): El valor debe ser verdadero/true
  • .toBeFalsy(): El valor debe ser falso/false
  • .toBeUndefined(): El valor debe ser indefinido/undifined
    • Nota: Tambien podemos negar las expreciones a comparar usando .not.Metodo()
